Proyectos, organizaciones y empresas utilizando noPoll

Wind River: usa noPoll

Wind River (http://www.windriver.com/) usa no noPoll para sus productos y dispositivos que requieren soporte para WebSocket.

Core-Admin: plataforma de administración de servidores

Core-Admin es una solución extensible para la gestión de servidores y la administración de servicios que integra en una única solución potentes paneles de administración, un sistema de comprobación altamente detallado tanto interno como externo y un sistema de notificación de eventos que le permitirán realizar un seguimiento al minuto del estado de todos los servidores y dispositivos conectados a la consola central.

Core-Admin usa noPolll para proporcionar soporte de BEEP sobre WebSocket al sistema.

Vea más detalles en la página del proyecto: http://www.core-admin.com/portal/es

MyQtt: OpenSource MQTT stack

MyQtt es un stack Open Source profesional del protocolo MQTT, escrito en ANSI C, que soporta las versiones v3.1.1 y v3.1, especializado en el soporte para crear servidores MQTT.

MyQtt usa noPoll para proporcionar soporte de MQTT sobre WebSocket

Vea más detalles en la página del proyecto: http://www.aspl.es/myqtt

stacom-software.de

Stacom Software Engineering ha usado la librería noPoll para integrar el soporte de websocket en un servicio web ejecutando en un dispositivo embebido linux. El soporte websocket es usado para proporcionar datos de estado para autenticar a los clientes. La autenticación es realizada usando certificados en el lado cliente.

Testimonio: “noPoll es fácil de integrar y se ajusta a las necesidades de dispositivos embebidos. Buen soporte profesional.”

Vea más detalles en: http://www.stacom-software.de

Vortex Library: implementación BEEP profesional

Vortex Library es una implementación de las definiciones estándar RFC 3080 / RFC 3081, conocidas como el protocolo BEEP, implementadas sobre la pila TCP/IP, usando el lenguaje C.

Incluye una implementación completa de XML-RPC sobre BEEP (soporte al RFC 3529), soporte completo para el perfil TUNNEL (lo que permite utilizar un proxy para todos los perfiles desarrollados sobre BEEP), soporte para proxies HTTP y mucho más.

Vea los detalles en la página del proyecto: http://www.aspl.es/vortex

Turbulence: servidor de aplicaciones BEEP

Turbulence es un servidor general BEEP construido sobre Vortex Library, que proporciona varias características listas para usar para facilitar el desarrollo y despliegue de aplicaciones BEEP.

Turbulence es extensible através de módulos (mod-tls, mod-sasl, mod-python, entre otros), y permite implementar perfiles de lado servidor que usados y combinados con otros perfiles através de la configuración.

Turbulence está utilizando noPoll para proporcionar un módulo listo para usar para configurar BEEP sobre WebSocket y los certificados asociados.

Vea más detalles en la página del proyecto: http://www.aspl.es/turbulence
